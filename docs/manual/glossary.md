**Block** — Blocks are objects that represent various elements of your tone, such as amps, cabs, effects, splits, loopers, inputs, outputs, and impulse responses; HX Stomp can accommodate up to eight simultaneous amp, cab, IR, effects, and/or looper blocks, DSP permitting.

**Model** — Each processing block can accommodate one model (or, in some cases, two models); HX Stomp includes over 80 guitar and bass amps, over 40 cabs, and over 220 effects models.

**Preset** — A preset is your tone; it consists of all blocks, snapshots, footswitch assignments, and controller assignments.

**Snapshot** — Snapshots are recalled per preset and allow you to select Snapshot 1, 2, or 3 to change block bypass states and parameter values without switching presets.

**Path** — A path is the stereo signal route through the preset; presets support either a single serial (stereo) path or two parallel stereo paths (A and B) that are split and then merged.

**DSP** — DSP (Digital Signal Processing) is the computing engine underlying HX Stomp; because some models require many times more DSP horsepower than others, you may eventually run out of DSP as you add blocks to a preset.

**IR** — IRs (Impulse Responses) are mathematical functions representing the sonic measurements of audio systems; HX Stomp can store up to 128 custom or third-party IRs at a time.

**Cab** — Cab blocks are speaker cabinet models used to color an amp signal; they come in Single (mono) and Dual (stereo, two separate Cab & Mic models) subcategories, with both IR-based and Legacy types available.

**Amp** — Amp blocks are identical to Amp+Cab blocks, except they contain no matched cab model.

**Preamp** — Preamp blocks are versions of each Amp model that model only the preamp stage of the amplifier circuit, require less DSP than a full Amp block, and include a studio microphone preamp within the Preamp > Mic category.

**Amp+Cab** — Amp+Cab blocks are convenient combined amp and cabinet blocks where selecting an amp model automatically pairs it with a matching cab model.

**Send/Return** — The HX Stomp stereo TRS Send and separate left and right Returns are used to insert your favorite stompboxes anywhere in the signal flow or for connecting to your guitar amp via 4-Cable Method.

**FX Loop** — FX Loop blocks appear alongside Send and Return blocks in the Send/Return category and allow external stompboxes to be inserted into the signal path at a specific point.

**Split** — Split blocks appear in presets with two parallel paths and mark the point where the single signal is divided into stereo path A (upper) and stereo path B (lower).

**Mixer** — Mixer blocks appear in presets with two parallel paths and mark the point where path A and path B are merged back together, with the ability to pan each path freely between outputs.

**Looper** — The Looper block provides on-board looping functionality; only one Looper block is allowed per preset.

**Controller** — Controllers are used to adjust various parameters in real time; for example, an external expression pedal can control wah, or a MIDI keyboard mod wheel can control delay feedback and reverb depth.

**Bypass** — Bypassing a block toggles it on and off (a bypassed block appears semi-transparent in Edit view); bypassing HX Stomp completely routes the signal through either Analog bypass (true bypass, no A/D/A conversion) or DSP bypass (delay echoes and reverb tails decay naturally).

**Mix/Blend** — Mix or Blend controls on effects blocks determine the balance between the dry (unaffected) signal and the wet (effected) signal; when an effect lacks these controls, placing it on a parallel path achieves the same result.

**TAP Tempo** — TAP Tempo allows you to set the global tempo in BPM by pressing FS3 (TAP/TUNER) repeatedly, and delay/modulation parameters set to note values will follow the tapped tempo or incoming MIDI clock.
